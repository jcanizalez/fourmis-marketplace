## Overview

API testing toolkit — REST API testing patterns (Supertest, httpx, Go httptest), mock server setup (MSW), contract testing (OpenAPI validation), API security auditing, HTTP client best practices, and API documentation. 6 skills, 3 commands, 1 agent. No dependencies.

## Skills

| Skill | Activates when... |
|-------|-------------------|
| `api-documentation` | When the user asks to generate API documentation |
| `api-security-testing` | When the user asks to test API security |
| `contract-testing` | When the user asks about API contract testing |
| `http-client` | When the user asks to build an HTTP client |
| `mock-servers` | When the user asks to create a mock API server |
| `rest-api-testing` | When the user asks to write REST API tests |

## Commands

| Command | Description |
|---------|-------------|
| `/api-test` | Run API tests against your endpoints — validates responses, status codes, headers, and schemas. Supports REST, GraphQL, and WebSocket APIs |
| `/http` | Make HTTP requests from the terminal — test endpoints, inspect responses, debug APIs with formatted output and timing |
| `/mock` | Set up mock API servers for development and testing — MSW handlers, fixture-based mocks, and API simulation with realistic delays and errors |

## Agents

### api-tester
Autonomous API testing agent — discovers endpoints, generates comprehensive test suites, sets up mock servers, validates contracts, and audits API security
